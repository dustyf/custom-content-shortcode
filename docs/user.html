User status

The [is] shortcode is used to display content based on user status.

Display user status

[is admin]You are an administrator.[/is] [is login]You are logged in.[/is] [is logout]You are logged out.[/is] [is user="john"]You are John.[/is] [is role="subscriber"]You are a subscriber.[/is] Available parameters:
  • admin - user has admin capability (manage options)
  • login - user is logged in
  • logout - user is logged out
  • user - user name or ID - multiple values are possible: "1, 3, 22"
  • role - user role - default roles are administrator, editor, author, contributor, or subscriber
  • capable - user capability, for example: capable="manage_options"
    • Multiple values are possible: role="admin, subscriber" will be true if the user is admin or subscriber. See the list of available user roles and capabilities under Dashboard -> Content -> User Roles.
You can also use [is not] to display something when the condition is not true.

Login and logout

The [url] shortcode is also used to display login and logout links.

Display a login link

<a href="[url login]">User login</a>

Display a logout link with redirect to home

<a href="[url logout go='home']">Logout</a> Available parameters:
  • login - login link
  • logout - logout link
  • go - redirect user after login/logout; use a full URL, or the name of a post or page; go="home" will return the user to the home page.
Here is an example using both [is] and [url] to show a login/logout link based on user status. [is logout]     <a href="[url login go='home']">         Login Button     </a> [/is] [is login]     <a href="[url logout go='somewhere']">         Logout Button     </a> [/is]

Redirect users

To redirect users to a different URL, use the [redirect] shortcode. As soon as it is executed, the user will be taken to another page.

Redirect if visitor is not logged in

[is not login]     [redirect]http://example.com/guest/[/redirect] [/is] Note that the URL is not a parameter, it is wrapped inside. This is to allow relative links. [is login]     [redirect][url site]/user-area/[/redirect] [/is] The [redirect] shortcode can be placed anywhere, but the best place is at the top of the page.
