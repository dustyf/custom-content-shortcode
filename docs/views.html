Content views

Here is an example of how to create templates to display your content. Let's imagine a bicycle shop. We create a custom post type called bicycle, and add custom fields such as model, price, and description. Add all bicycles in the store as new entries, with featured image and other info in the fields. Create a new page to display the bicycles. A basic template could be: [loop type="bicycle"]     [field image]     Model: [field model]     Price: [field price]     Description: [field description] [/loop] Another page can display a list of bicycles in the freestyle category: Freestyle bikes available: <ul>     [loop type="bicycle" category="freestyle"]         <li>             [field title-link]             [field model] - [field price]         </li>     [/loop] </ul>

Editing templates

You can create and edit these templates inside the post itself, with the Text editor. There are a couple points to keep in mind:
  • WordPress automatically formats post content, adding paragraph tags and break lines. If you find that there are unnecessary tags inserted inside the [loop], try enabling Move post content formatting under Settings.
  • Don't switch to the Visual editor when using HTML, as the code can sometimes be wrongly interpreted. To solve this, you can selectively disable the visual editor with the plugin Raw HTML, or move the code outside the post content with one of the methods below.
The template can also be loaded from a number of places.
  • Custom post type: [content type="template" name="home-page"]
  • Custom field: [content field="code_block"]
  • File: [load dir="views" file="recent-posts.html"]
  • Sidebar: Enable shortcodes inside Text widget in Settings
