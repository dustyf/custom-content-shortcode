Basic Examples

Here are some examples of using the [content] shortcode.

Display post content by name

[content name="hello-world"]

Display the featured image of a page

[content type="page" name="about-me" field="image"]

Display a custom field from a custom post type

[content type="apartment" name="lux-suite-22" field="rent-per-day"]

Display fields from the current post

The Article <b>[content field="title"]</b> by [content field="author"] was written on [content field="date"].

Parameters

Available parameters of the [content] shortcode:
  • type - which post type to target: post, page, or custom post type - the default is any post type
  • name, title, or id -  which post to target: name/slug, title, or ID - the default is the current post
  • field - which field to display - the default is the main content of the post. You can display custom fields you created, as well as predefined fields: title, id, author, date, url, image, image-url, thumbnail, and excerpt.
In addition:
  • format - add paragraph tags <p> and preserve line breaks <br> - set to true/false - by default, post content is formatted and for all other fields, it's set to false
  • class - add a <div> class to the output for styling purpose
  • words or length/len- limit number of words or characters
  • date_format - use a custom format when displaying field="date" - for example, "d.m.Y" will display as 18.11.2013. Use double backslashes to escape characters - "Y/m/d \\a\\t g:i A" will show as 2013/11/17 at 11:06 PM. For more info, see the Codex: Formatting Date and Time.
For other types of content:
  • area or sidebar - display a widget area/sidebar by title
  • menu - display a menu list by name/slug, title, or ID. Use the ul parameter for Bootstrap nav-tabs, nav-pills and nav-pills-stacked. For a Bootstrap navbar menu, please see the section for the [navbar] shortcode.
  • image - display image fields by name/slug
